jist.in

This is an (almost) server-less tool to display GitHub Gists for comfortable reading.
It processes gist contents entirely by client-side JavaScript in users' browsers.

Usage

For simplest cases, you can
  1. Create a gist with a Markdown file. The gist can be either public or private.
  2. Copy the gist's numerical ID gist_id from the page URL, which looks like
    • 5794706 for public gists, or
    • 908e37c83da28b50a897 for private ones.
  3. Comfortably read at http://jist.in/gist_id.
    You can check out jist.in/5855373 for an example post.
Or you can use the Chrome extension or this jist.in bookmarklet on the gist page to automatically jump to its corresponding jist.in page.

Features

Besides Markdown, jist.in also supports:
  • Local usage (completely server-less),
  • Custom CSS (check out the example),
  • Custom JavaScript (example from three.js),
  • HTML content (Mike Bostock’s epicyclic gearing),
  • LaTeX for mathematical content (a short bio of Évariste Galois),
  • List of gists with specific tags, e.g. blog (zhuzhuor.jist.in/jist-blog/),
  • Custom domain name (jist-blog.zhuzhu.org).
For detailed usage instructions, please refer to the GitHub repo.
